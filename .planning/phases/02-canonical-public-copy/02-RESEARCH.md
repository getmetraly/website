## Findings

- The most important public fixes are license/pricing and unsupported readiness claims.
- `pricing.html` should become the canonical source for public tier copy in the website.
- `terms.html` should not attempt to fully define software licensing; it should refer to the repository license and website/demo terms.
- `index.html` can keep strong positioning, but should replace FSL and "production-grade day one" style claims with status-aware language.
- The demo CTA can remain, but the login page should describe the current state as a synthetic product tour or preview until Phase 3 builds the real demo.
- `docs.html` has major issues but is explicitly Phase 4, so Phase 2 should only fix cross-page/footer links if they are touched elsewhere.

## Plan Split

- `02-01`: pricing and legal pages.
- `02-02`: home page positioning/feature-readiness.
- `02-03`: cleanup of unsupported proof points, quickstart/footer/demo wording.
