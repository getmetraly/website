# Codebase Structure

**Analysis Date:** 2026-05-05

## Directory Layout

```text
website/
├── .git/                 # Separate git repository metadata
├── .planning/            # GSD planning artifacts for this website project
│   └── codebase/         # Codebase map documents
├── index.html            # Main marketing page
├── pricing.html          # Pricing and license page
├── docs.html             # Static documentation page
├── login.html            # Static demo login/mock entry page
├── privacy.html          # Privacy policy page
├── terms.html            # Terms of service page
└── theme.js              # Shared light/dark theme runtime
```

## Directory Purposes

**Project root:**
- Purpose: All current website source files live at the top level.
- Contains: standalone HTML pages and one shared JavaScript file.
- Key files: `index.html`, `pricing.html`, `docs.html`, `login.html`, `theme.js`.
- Subdirectories: `.planning/` only for GSD artifacts after this mapping.

**.planning/codebase/**
- Purpose: Repository analysis used by GSD planning.
- Contains: `STACK.md`, `INTEGRATIONS.md`, `ARCHITECTURE.md`, `STRUCTURE.md`, `CONVENTIONS.md`, `TESTING.md`, `CONCERNS.md`.
- Key files: all generated by `$gsd-map-codebase`.

**Adjacent repositories:**
- `../app/` - Real Metraly application repository with Go API, React/Vite UI, Docker Compose, and existing `.planning`.
- `../docs/` - Canonical product/strategy/legal/technical documentation repository.
- These are not child directories of `website`, but this website project must use them as source inputs.

## Key File Locations

**Entry Points:**
- `index.html`: Primary marketing/site home.
- `pricing.html`: Pricing/license page.
- `docs.html`: Documentation page.
- `login.html`: Demo entry page.

**Configuration:**
- No `package.json`, `.env`, bundler config, lint config, or deploy config exists in `website`.
- `theme.js`: only shared runtime configuration-like behavior.
- `../app/.planning/config.json`: GSD settings copied conceptually for this project.

**Core Logic:**
- `theme.js`: shared theme persistence and toggle behavior.
- `login.html`: contains `handleLogin(event)` mock login behavior.
- `index.html`: contains page-local JavaScript and a reference to missing `tweaks-panel.jsx`.

**Testing:**
- No test files exist in `website`.
- Adjacent app tests and UI code live in `../app`.

**Documentation:**
- `docs.html`: rendered static docs page in the website.
- `../docs/`: actual canonical documentation source.
- `.planning/codebase/*.md`: internal planning/reference docs.

## Naming Conventions

**Files:**
- Lowercase simple names for pages: `index.html`, `pricing.html`, `docs.html`, `login.html`.
- Shared JavaScript uses lowercase: `theme.js`.
- Future page files should stay lowercase kebab-case if static HTML remains.

**Directories:**
- Hidden `.planning/` for GSD artifacts.
- No application source directories currently exist.

**Special Patterns:**
- Page-specific CSS is embedded in each HTML file.
- Shared theme variables use CSS custom properties under `:root`, `[data-theme="dark"]`, and `[data-theme="light"]`.
- Global JS entry is exposed as `window.MetralyTheme`.

## Where to Add New Code

**New static page:**
- Add `{page-name}.html` at repo root if keeping the current architecture.
- Duplicate or extract shared navigation/footer/theme styles intentionally.
- Link from relevant nav/footer sections.

**Shared styling:**
- Prefer adding a shared CSS file before expanding the site further.
- Existing duplicated variable definitions across pages should be consolidated.

**Demo mode:**
- Do not extend `login.html` as a fake login only.
- Add a real demo route/page or introduce a build system that can embed parts of `../app/ui`.
- Candidate source modules: `../app/ui/src/features/*`, `../app/ui/src/types/mocks/*`, `../app/ui/src/api/mockApi.ts`.

**Docs sync:**
- Add a generation/import process or a documented manual sync path from `../docs`.
- Start with `../docs/STATUS.md`, `../docs/strategy/monetization-v2.md`, `../docs/strategy/positioning-uvp.md`, `../docs/legal/legal-risks-and-compliance.md`, and `../docs/product/onboarding.md`.

**Tests:**
- If static-only: add HTML/link/accessibility checks and Playwright smoke tests.
- If bundled: add package scripts and browser tests alongside source structure.

## Special Directories

**.planning/**
- Purpose: GSD project state and planning documents.
- Source: generated/maintained by GSD workflows.
- Committed: yes, per user preference to use `../app` defaults with `commit_docs: true`.
